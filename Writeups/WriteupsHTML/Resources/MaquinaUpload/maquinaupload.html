Writeup: Upload

Introducción

En este writeup exploraremos la resolución de la máquina Upload de DockerLabs. La máquina tiene una dificultad facil.

Reconocimiento

Comenzamos realizando un escaneo básico con nmap:

$sudo nmap -p- --open -sS --min-rate 5000 -vvv -Pn -n 172.17.0.2 

    PORT   STATE SERVICE REASON
    80/tcp open  http    syn-ack ttl 64
    MAC Address: 02:42:AC:11:00:02 (Unknown)
                    
    Read data files from: /usr/bin/../share/nmap
    Nmap done: 1 IP address (1 host up) scanned in 3.22 seconds
    Raw packets sent: 65536 (2.884MB) | Rcvd: 65536 (2.621MB)

El resultado revela los siguientes puertos abiertos:

  • 80/tcp: Servidor http ttl64 (linux)

Análisis de Vulnerabilidades

Al inspeccionar el servidor web, encontramos una página para subir archivos:

Esto nos lleva a analizar las rutas ocultas:

gobuster dir -u http://10.10.10.123 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

El análisis revela una carpeta interesante:

  • /admin: Panel de administración

Explotación

Usamos sqlmap para verificar vulnerabilidades de inyección SQL en el formulario de inicio de sesión del panel de administración:

sqlmap -u "http://10.10.10.123/admin/login.php" --forms --dbs

El resultado confirma que el formulario es vulnerable. Usamos las credenciales obtenidas para acceder al panel:

  • Usuario: admin
  • Contraseña: password123

Escalada de Privilegios

Tras obtener acceso SSH con las credenciales obtenidas, verificamos el sistema en busca de privilegios elevados:

sudo -l

Esto revela que podemos ejecutar /bin/bash como root sin contraseña:

sudo /bin/bash

¡Esto nos da acceso root completo!

Conclusión

La máquina Inventada es un excelente ejemplo de cómo la enumeración adecuada y el análisis de vulnerabilidades pueden llevar a una explotación exitosa. ¡Gracias por leer este writeup!
